## Introduction

With Cloud Portal now connected, the next step is to extend SSO to **Sitecore Content Hub**.  
Unlike Cloud Portal, Content Hub manages its SAML configuration directly through the **Portal Configuration JSON** — this is where you define the identity provider metadata, enable the SAML module, and map incoming Azure `groups` claims to Content Hub roles.

This article walks through the sections of the JSON you’ll need to modify, along with example placeholders you can replace with your own Azure Group Object IDs and Content Hub groups.

Before you start, make sure you have:
- The **Content Hub Federation Metadata XML** exported from Azure → `[Content Hub Metadata File]`
- Your **Azure Group Object IDs** and the **target Content Hub groups** they should map to → `[Group Mapping Table]`
- The **Content Hub Admin** responsible for applying and validating the change → `[Content Hub Admin]`

---

## 1. Access Portal Configuration

1. Sign in to **Content Hub** as an administrator (`[Content Hub Admin Account]`).
2. Navigate to **Manage → Settings → Portal Configuration**.
3. Switch to **Text View** or **Code View** to access and edit the JSON directly.

> **Tip:** Always take a backup of your existing Portal Configuration JSON before making any edits — it’s your safety net if something goes wrong.

---

## 2. Update the JSON (Using Placeholder Values)

Find the `ExternalAuthenticationProviders` section in the JSON.  
You’ll either add or modify a `saml` entry to include your Azure Entra ID details.
